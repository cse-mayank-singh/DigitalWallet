Digital Wallet Backend

Endpoint Guide and Test Recipes

This page lists the public API routes exposed through the API gateway and the internal service-to-service routes used by the microservices. All examples are aligned with the controller signatures currently in the codebase.

Recommended Base URL

Use the API gateway first unless you are debugging a single service.

Gateway: http://localhost:8080

Direct Service Ports

auth-service   : http://localhost:8084
user-service   : http://localhost:8082
wallet-service : http://localhost:8083
rewards-service: http://localhost:8085
eureka-server  : http://localhost:8761

Auth Model

Protected routes go through gateway JWT filtering. After login, keep the bearer token and the returned user id and role.

How To Test Quickly

1. Bootstrap admin

The auth service now seeds an admin on first run if no admin exists.

Email   : admin@digitalwallet.local
Phone   : 9999999999
Password: Admin@123

2. Get a token

Login and copy the returned accessToken.

POST http://localhost:8080/api/auth/login

3. Call secured routes

Authorization: Bearer <accessToken>

Gateway injects downstream headers like X-User-Id and X-User-Role.

Public routes: all /api/auth/** endpoints. Protected routes through gateway: /api/users/**, /api/kyc/**, /api/admin/**, /api/wallet/**, /api/payment/**, /api/rewards/**.

Common Test Snippets

# Signup a normal user
curl -X POST http://localhost:8080/api/auth/signup ^
  -H "Content-Type: application/json" ^
  -d "{\"fullName\":\"Amit Kumar\",\"email\":\"amit@example.com\",\"phone\":\"9876543211\",\"password\":\"Password@1\"}"

# Login as admin
curl -X POST http://localhost:8080/api/auth/login ^
  -H "Content-Type: application/json" ^
  -d "{\"email\":\"admin@digitalwallet.local\",\"password\":\"Admin@123\"}"

# Protected request pattern
curl http://localhost:8080/api/users/profile ^
  -H "Authorization: Bearer <ACCESS_TOKEN>"

Authentication Endpoints

POST/api/auth/signup

Create a normal user account.

{
  "fullName": "Amit Kumar",
  "email": "amit@example.com",
  "phone": "9876543211",
  "password": "Password@1"
}

POST/api/auth/login

Email/password login. Returns access token, refresh token, user id and role.

{
  "email": "admin@digitalwallet.local",
  "password": "Admin@123"
}

POST/api/auth/login/phone

Phone/password login.

{
  "phone": "9999999999",
  "password": "Admin@123"
}

POST/api/auth/send-otp

Send login OTP to email.

{
  "email": "amit@example.com"
}

POST/api/auth/verify-otp

Verify login OTP and receive tokens.

{
  "email": "amit@example.com",
  "otp": "123456"
}

POST/api/auth/forgot-password/send-otp

Start password reset flow.

{
  "email": "amit@example.com"
}

POST/api/auth/forgot-password/verify-otp

Verifies OTP and returns a temporary resetToken.

{
  "email": "amit@example.com",
  "otp": "123456"
}

POST/api/auth/reset-password

Use the token from the previous step.

{
  "resetToken": "paste-reset-token-here",
  "newPassword": "NewPassword@1"
}

POST/api/auth/refresh

{
  "refreshToken": "paste-refresh-token-here"
}

POST/api/auth/logout

{
  "refreshToken": "paste-refresh-token-here"
}

GET/api/auth/health

Simple health check.

User and KYC Endpoints

GET/api/users/profile

Get current user profile.

curl http://localhost:8080/api/users/profile ^
  -H "Authorization: Bearer <ACCESS_TOKEN>"

PUT/api/users/profile

Update name and/or phone.

{
  "name": "Amit Kumar Updated",
  "phone": "9876543212"
}

POST/api/kyc/submit

Multipart form-data endpoint. Use docType, docNumber, optional docFile.

curl -X POST "http://localhost:8080/api/kyc/submit" ^
  -H "Authorization: Bearer <ACCESS_TOKEN>" ^
  -F "docType=AADHAR" ^
  -F "docNumber=123412341234" ^
  -F "docFile=@C:\path\to\document.jpg"

GET/api/kyc/status

Fetch current user KYC status.

Wallet and Payment Endpoints

GET/api/wallet/balance

Current wallet balance.

POST/api/wallet/transfer

Transfer money from current user to another user.

{
  "receiverId": 2,
  "amount": 250.00,
  "idempotencyKey": "tx-20260330-001",
  "description": "Lunch split"
}

POST/api/wallet/withdraw

{
  "amount": 100.00
}

GET/api/wallet/transactions?page=0&size=10

Paginated transaction history.

GET/api/wallet/ledger?page=0&size=20

Ledger entries.

GET/api/wallet/statement?from=2026-03-01&to=2026-03-30

JSON statement for a date range.

GET/api/wallet/statement/download?from=2026-03-01&to=2026-03-30

Downloads CSV statement.

POST/api/payment/create-order?amount=500

Creates a Razorpay order for top-up. Amount is a query parameter, not JSON body.

curl -X POST "http://localhost:8080/api/payment/create-order?amount=500" ^
  -H "Authorization: Bearer <ACCESS_TOKEN>"

POST/api/payment/verify

Verify Razorpay payment and credit wallet.

{
  "razorpayOrderId": "order_xxx",
  "razorpayPaymentId": "pay_xxx",
  "razorpaySignature": "signature_from_checkout"
}

Gateway Fallback Endpoints

These are diagnostic endpoints on the API gateway. They return 503-style fallback payloads when circuit breakers are triggered.

Route Purpose
/fallback/authAuth-service fallback
/fallback/userUser-service fallback
/fallback/walletWallet-service fallback
/fallback/rewardsRewards-service fallback
/fallback/adminAdmin fallback
